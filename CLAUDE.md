## Project Overview

Personal portfolio site for Ethan Kennedy, deployed as a Cloudflare Worker at ethankennedy.uk. Server-side rendered using Hono with JSX (not React — uses `hono/jsx`), styled with Tailwind CSS + DaisyUI, with client-side interactivity via htmx and Three.js.

## Commands

- **Dev server:** `npm run dev` (runs wrangler dev with live reload)
- **Deploy:** `npm run deploy` (deploys to Cloudflare Workers)
- **Build Tailwind:** `npm run tailwind` (watches and compiles `tailwind.css` → `assets/static/tailwind.css`)
- **Generate routes:** `npm run routes` (regenerates `src/router.ts` from `src/routes/`)

When developing, run both `npm run dev` and `npm run tailwind` in separate terminals.

## Architecture

- **`src/main.ts`** — Hono app entrypoint. Serves static assets from `assets/` and applies the root HTML layout to all routes.
- **`src/layout.tsx`** — Root HTML shell (head, fonts, scripts). Uses `hono/jsx-renderer`.
- **`src/router.ts`** — Auto-generated by `hono-router` from files in `src/routes/`. Do not edit manually.
- **`src/routes/`** — File-based routing. Each file exports `onRequestGet`/`onRequestPost` handlers.
- **`src/components/`** — JSX components rendered server-side. These are Hono JSX, not React — no hooks, no state, no client-side hydration.
- **`assets/static/`** — Static files served at `/static/*`. Includes compiled Tailwind CSS and client-side JS (htmx, Three.js hero animation, scroll animations).
- **`tailwind.css`** — Tailwind source with custom component classes (`btn-primary`, `card`, `tag-chip`, etc.).
- **`tailwind.config.js`** — Custom design tokens: dark theme colors (`bg`, `surface`, `accent-blue`, `accent-violet`), typography scales, animations.

## Key Conventions

- JSX uses `class` not `className` (Hono JSX, not React)
- TypeScript with `jsxImportSource: "hono/jsx"` — do not import React
- Static asset type declaration in `src/custom.d.ts` for `__STATIC_CONTENT_MANIFEST`
- The `wrangler.toml` `account_id` is not secret (Cloudflare account IDs are public)
